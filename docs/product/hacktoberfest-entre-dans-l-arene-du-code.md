## La constitution Belge

[

GitHub - geektortoise/La-Constitution-Belge: La Constitution Belge

La Constitution Belge. Contribute to geektortoise/La-Constitution-Belge development by creating an account on GitHub.

![](https://github.githubassets.com/assets/pinned-octocat-093da3e6fa40.svg)GitHubgeektortoise

![](https://opengraph.githubassets.com/f51b56aa1baf76eb4065b163941b4e9d8eb24f5ceffa8cc7cc37eb61f5c2f720/geektortoise/La-Constitution-Belge)

](https://github.com/geektortoise/La-Constitution-Belge?ref=sfeir.dev)

- **technologie** : [Markdown](https://www.sfeir.dev/markdown-definition/)
- **owner** : [Renaud Hoyoux](https://www.sfeir.dev/author/geektortoise/)

Dans ce projet, [Renaud Hoyoux](https://www.sfeir.dev/author/geektortoise/) a pris le temps d'encoder _La Constitution Belge_ [au format Markdown](/definition/markdown-definition/)afin de la rendre disponible au plus grand nombre et de tracer l'évolution de cette dernière dans le temps.

Cela permet de replonger facilement dans le passé et de comprendre les évolutions de ce qui est considéré aujourd'hui comme fondamental.

## Composant customisé pour Vaadin

[

GitHub - ErwanLT/vaadin-custom-component: custom components for Vaadin projects

custom components for Vaadin projects. Contribute to ErwanLT/vaadin-custom-component development by creating an account on GitHub.

![](https://github.githubassets.com/assets/pinned-octocat-093da3e6fa40.svg)GitHubErwanLT

](https://github.com/ErwanLT/vaadin-custom-component?ref=sfeir.dev)

- **technologie** : [Java](/back/java/il-etait-une-fois-java/)

Vaadin propose de nombreux composants tant gratuits que payants. Malheureusement certains sont à créer soi-même puisqu'ils sont encore inexistants dans ce framework. Ce sont ces composants manquants que je propose dans ce projet.

## **Exporter tous vos schémas Draw.io facilement**

[

GitHub - rlespinasse/drawio-export: Export Draw.io diagrams using docker

Export Draw.io diagrams using docker. Contribute to rlespinasse/drawio-export development by creating an account on GitHub.

![](https://www.sfeir.dev/content/images/icon/pinned-octocat-093da3e6fa40.svg)GitHubrlespinasse

![](https://www.sfeir.dev/content/images/thumbnail/drawio-export)

](https://github.com/rlespinasse/drawio-export?ref=sfeir.dev)

- **technologies** :
    - [Docker](https://www.sfeir.dev/tag/docker/)
    - [Rust](https://www.sfeir.dev/front/et-si-rust-etait-prod-ready-pour-le-developpeur-ts/)
    - [GitHub Action](https://www.sfeir.dev/rex-certification-github-actions/)
- **owner** : [Romain Lespinasse](https://www.linkedin.com/in/romain-lespinasse/?originalSubdomain=fr&ref=sfeir.dev)
- **contributeurs** :
    - [Nicolas Hansse](https://www.linkedin.com/in/nicolas-hansse/?ref=sfeir.dev)
    - [Gaetan BOGAERT](https://www.sfeir.dev/author/gaetan/)

Vous utilisez [draw.io](https://app.diagrams.net/?ref=sfeir.dev) pour vos schémas, mais trouvez fastidieux de les exporter manuellement? Dans ce projet, [Romain Lespinasse](https://www.linkedin.com/in/romain-lespinasse/?originalSubdomain=fr&ref=sfeir.dev) a mis en place une image docker pour exporter en masse vos schémas sous Draw.io.

[Une GitHub Action](https://www.sfeir.dev/rex-certification-github-actions/) est disponible pour automatiser cet export:

## L'article s'achève, mais la fête commence !

Hacktoberfest est bien plus qu'un simple événement : c'est une célébration de la collaboration et de l'innovation dans l'univers de l'open source.  
Que vous soyez un vétéran ou que vous fassiez vos premiers pas, chaque contribution compte et aide à renforcer des projets qui bénéficient à tous.  
Alors, plongez dans cette aventure, apprenez, partagez, contribuez à l'édifice, et surtout, amusez-vous !
